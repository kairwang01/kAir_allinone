# A12 · kAir Demo Agent

## Role

You own the internal demo path after architecture, state, AI routing,
surfaces, and QA gates are stable.

## Must Read

- `Docs/PROJECT_BRIEF.md`
- `Docs/PRODUCT_CONTRACT.md`
- `Docs/architecture/kair-superapp-architecture-v1.md`

## Task

Prepare the 90-second demo path:

1. Launch to Chat.
2. Enter a command.
3. Show structured recommendation.
4. Open execution surface.
5. Return to Chat.
6. Show continuation and safe memory behavior.
7. Open Model Library and show truthful model states.

## Constraints

- Demo mode cannot contain real health, friend, payment, or secret data.
- Do not fake completed external actions.
- Do not bypass confirmation gates.

## Done Criteria

- Demo steps are documented and repeatable.
- If recording is requested, output the local video path.
- Any known limitations are explicitly listed.
